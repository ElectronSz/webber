Welcome to Tornado Web Server!

This is the default welcome page for your Go-powered Tornado web server.

Server Features

  • Logging Middleware: Every request is logged, showing method, URI, remote address, and completion time.
  • Rate Limiting: Protects against abuse by limiting requests per client IP. Configurable via `RateLimitRPS` and `RateLimitBurst`.
  • Gzip Compression: Automatically compresses responses for supported clients, reducing bandwidth usage and improving load times.
  • URL Rewriting: Demonstrates how to redirect or rewrite URLs (e.g., `/old` to `/new`). Try /old/example.
  • Reverse Proxy & Load Balancing: Distributes incoming requests across multiple backend servers using a round-robin strategy. See /proxy.
  • WebSocket Support: Provides a dedicated endpoint for real-time, bidirectional communication. Access it at /ws.
  • Static File Serving with Caching: Efficiently serves static assets from the configured directory (`/static/`) with in-memory caching based on `CacheTTLSeconds`.
  • HTTP/2 and TLS: Secure and high-performance communication enabled by default.

Configuration

Tornado Web Server is configured via a config.json file in the same directory as the executable. 

{
  "port": "443",           // Port for the server to listen on (e.g., "80" for HTTP, "443" for HTTPS)
  "static_dir": "./static",   // Directory to serve static files from (e.g., "index.html")
  "proxy_targets": [         // List of backend URLs for the reverse proxy
    "http://localhost:8081",
    "http://localhost:8082"
  ],
  "rate_limit_rps": 10.0,    // Requests per second allowed per client IP
  "rate_limit_burst": 20,    // Burst capacity for the rate limiter
  "cache_ttl_seconds": 300   // Time-to-live for static file cache entries in seconds
}
